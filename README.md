# 🏥 Hospital Management System Analysis (SQL Project)

## 📌 Project Overview
A relational database project built using MySQL to manage and analyse 
hospital data including patients, doctors, and billing records.
This project demonstrates real-world SQL skills used in Data Analytics.

## 🛠️ Tools Used
- MySQL (via DB Fiddle)
- SQL (DDL + DML + Queries)
- GitHub (version control)

## 🗄️ Database Structure
3 linked tables built using Primary Keys and Foreign Keys:

| Table | Description |
|---|---|
| Doctors | Stores doctor details and specialization |
| Patients | Stores patient details linked to doctors |
| Billing | Stores billing records linked to patients |

##  Business Insights Generated

| Query | Insight |
|---|---|
| SELECT + WHERE | Filtered patients by gender |
| GROUP BY + COUNT | Identified most common diseases |
| SUM | Calculated total hospital revenue |
| JOIN + GROUP BY | Analysed doctor wise revenue |
| JOIN + WHERE | Tracked pending payments |

##  Key Findings
- Total hospital revenue: **₹72,500**
- Highest earning doctor: **Dr. Rao — ₹28,000**
- Most common disease: **Heart Disease — 2 patients**
- Pending payment: **Arjun Singh — ₹24,000**

##  SQL Concepts Used
- CREATE TABLE with PRIMARY KEY and FOREIGN KEY
- INSERT INTO
- SELECT, WHERE, ORDER BY
- GROUP BY with COUNT and SUM
- INNER JOIN across 3 tables
- Aggregate Functions
